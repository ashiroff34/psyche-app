# IP Assignment Agreements

**Why this directory exists:** IP assignment agreements are the #1 deal killer in tech acquisitions. Every person who contributed code, design, writing, or content to Thyself must have a signed agreement assigning their work to the company. Acquirers check this in diligence — missing agreements block or kill deals.

---

## Who needs an agreement?

- **Freelancers** who built features, wrote content, or created design assets
- **Contractors** paid via invoice for any Thyself work
- **Collaborators** who contributed code via GitHub (even 1 commit)
- **Content creators** who wrote type descriptions, lesson copy, or daily observations
- **Illustrators / designers** who created any visual assets

**NOT needed for:**
- You (Arianna) — as sole founder you own your own work
- Work done by Claude / AI tools — not copyrightable in most jurisdictions
- Third-party open-source libraries — covered by their licenses (but document the licenses separately)

## Before the next hire or collaboration

Before any new person contributes to Thyself:
1. Send them the TEMPLATE.md (or a DocuSign version of it)
2. Get it signed BEFORE they start work — not after
3. File the signed copy here immediately

Getting a signature after work is complete is legally harder and signals disorganization to acquirers.

---

## Diligence preparation

When an acquirer's legal team requests IP documentation:
1. Provide all signed agreements from this directory
2. Provide your own founder IP assignment (if company is ever incorporated — advisable before any deal)
3. Include a list of all open-source licenses used (see `package.json` → `npm audit` output)
